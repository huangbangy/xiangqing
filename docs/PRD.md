# 产品需求文档

## 1. 项目名称

新化相亲小程序

## 2. 目标

做一个面向新化地区的微信相亲小程序，先上线一个最小可用版本，跑通“注册 - 审核 - 浏览 - 联系 - 举报”这条主流程。

## 3. 目标用户

- 新化地区本地单身用户
- 在新化工作、生活，愿意找本地对象的用户
- 18岁及以上成年人

## 4. MVP范围

### 必做功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 微信授权登录 | 使用微信登录进入小程序 | 高 |
| 基础资料填写 | 性别、年龄、所在地、职业、婚姻状态等 | 高 |
| 头像和照片上传 | 至少1张头像，支持多张相册照片 | 高 |
| 资料审核 | 用户资料提交后由管理员审核 | 高 |
| 本地筛选浏览 | 按性别、年龄、地区、婚姻状态等筛选 | 高 |
| 首页今日推荐 | 按同城、年龄接近、生活标签、资料完整度做轻量推荐 | 中 |
| 推荐反馈 | 支持换一批、不感兴趣、恢复推荐池 | 中 |
| 资料详情页 | 展示完整个人资料 | 高 |
| 收藏/喜欢 | 记录用户关注对象 | 中 |
| 最近浏览 | 记录最近查看过的资料，方便回看 | 低 |
| 联系申请 | 向对方发起联系申请 | 高 |
| 联系方式保护 | 双方进入同意联系状态后才展示微信号/手机号 | 高 |
| 基础站内聊天 | 联系申请通过后生成会话，支持文字消息和快捷开场 | 中 |
| 资料完整度 | 在“我的”页提示资料补全程度和缺失项 | 中 |
| 三句话认识我 | 用短句展示生活节奏、关系期待、周末习惯 | 中 |
| 生活标签 | 用轻标签表达性格和生活方式 | 中 |
| 家长代看模式 | 家长可代替成年子女浏览并发起家长沟通 | 中 |
| 举报/拉黑 | 处理违规用户与内容 | 高 |
| 我的页面 | 查看资料状态、收藏、申请记录 | 中 |
| 后台管理 | 审核资料、处理举报、封禁账号 | 高 |

### 暂不做

- 实时长连接聊天、已读回执、语音视频
- 短视频
- 动态广场
- 直播
- 付费会员
- 红娘撮合结算
- 复杂自动推荐算法

## 5. 核心流程

1. 用户微信登录
2. 填写个人资料并上传照片
3. 提交审核
4. 审核通过后进入公开列表
5. 用户查看今日推荐、换一批、标记不感兴趣，或浏览并筛选合适对象
6. 发起收藏或联系申请
7. 对方同意后展示联系方式，并生成站内聊天会话
8. 发现问题可举报，管理员处理

## 6. 页面范围

- 首页/列表页
- 筛选页
- 个人资料详情页
- 资料编辑页
- 我的页
- 收藏列表页
- 联系申请列表页
- 消息列表页
- 聊天详情页
- 举报页
- 管理后台

## 7. 资料字段建议

- 微信昵称
- 头像
- 性别
- 年龄/生日
- 所在地区
- 家乡
- 身高
- 学历
- 职业
- 收入区间
- 婚姻状态
- 是否有小孩
- 住房情况
- 车辆情况
- 自我介绍
- 择偶要求
- 联系方式（仅在合适流程下展示）

## 8. 审核规则

- 必须成年人
- 头像和资料不能明显造假
- 不能出现广告、引流、诈骗、色情、违法内容
- 联系方式默认不直接公开
- 微信号和手机号只在联系申请同意后展示
- 家长模式仅面向成年子女，不替代本人授权
- 管理员可通过、驳回、下架、封禁

## 9. 运营原则

- 先招募一批真实种子用户
- 资料不完整的用户不进入公开列表
- 举报必须可追踪
- 管理后台必须保留审核记录

## 10. 成功标准

- 用户能顺利注册并提交资料
- 管理员能完成审核
- 用户能正常浏览和筛选资料
- 用户能发起联系申请
- 举报和封禁流程可用
